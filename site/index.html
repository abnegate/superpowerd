superpowerd

Run 6+ Claude Code agents in parallel across repos. Never stop for rate limits.

Pool your Claude Max accounts. When one hits a limit, superpowerd rotates to the next — automatically. Full OAuth flow, browser cookie import, every terminal re-authenticated. Your team of agents keeps shipping while the account cools down.

curl -fsSL https://superpowerd.jakebarnby.com/install | bash
Why
6+
Parallel agents
Each repo gets its own WezTerm pane running Claude Code. Work across your entire codebase simultaneously — backend, frontend, infra, SDKs, all at once.
0s
Downtime on limits
Rate limit hits trigger automatic rotation to the next account. Browser automation handles the OAuth flow. Your agents never see the "please wait" screen.
3
Accounts, one pool
Configure as many Claude Max accounts as you have. Round-robin rotation with 5-minute cooldown. Each account's limit resets independently.
1
Command to install
Clones the repo, installs WezTerm + Playwright, configures pane grid, starts the rate limit monitor as a system service, and imports your browser cookies. One curl pipe.
How it works
Workspace
WezTerm opens a 2-column grid with one pane per repo. Each pane launches Claude Code. Pane titles update live with repo · branch * · PR #123. Layout scales from 2 to 10+ panes with tuned split ratios. One keyboard shortcut per pane.
Detection
A background daemon tails Claude's debug logs. Watches for HTTP 429, 529, "rate limited", "overloaded", "usage limit". Filters out transient startup bursts. When a real limit hits, rotation triggers within seconds.
Rotation
Full automated OAuth flow via Playwright. Signs out of claude.ai, selects the next Google account, completes consent. Intercepts the claude auth login URL via a BROWSER env trap and finishes the callback in the same browser session. Then sends /login to every WezTerm pane.
Cookies
Zero-setup auth — imports cookies from your real browser. Firefox: reads cookies.sqlite directly. Chrome on macOS: decrypts via Keychain (PBKDF2 + AES-128-CBC, prompts once). Chrome on Linux: reads unencrypted. No manual login unless you have neither browser.
Dashboard
React/TypeScript web UI at localhost:3848. Active account indicator, CLI auth status, monitor state. One-click rotation buttons, live log stream via SSE. Start/stop the monitor from the browser.
Recovery
Claude Code slash command /auto-updater. Five recovery procedures for when auth breaks: CLI reset, browser session cleanup, credential nuke, selector updates, terminal re-auth.
Rotation flow
Rate limit detected
Monitor daemon catches 429/529 in ~/.claude/debug/*.txt.
CLI logout
claude auth logout clears current credentials.
Browser switch
Playwright signs out of claude.ai and into the next Google account.
OAuth interception
claude auth login URL captured via BROWSER env trap, completed in Playwright.
Callback
OAuth redirects to localhost. CLI receives the token.
All terminals re-authenticated
/login sent to every WezTerm pane. Desktop notification. All agents resume.

web=w
infra=i
