superpowerd

Multi-account Claude Code workspace manager. Rate limit detection, automatic account rotation, WezTerm pane grid, real-time dashboard. Never wait for limits again.

curl -fsSL https://superpowerd.jakebarnby.com/install | bash
What it does
Account
rotation
Monitors Claude's debug logs for 429s, rate limit messages, and overload signals. When detected, rotates to the next account automatically. Handles the full OAuth flow — signs out of claude.ai, switches Google account, re-authenticates the CLI, and sends /login to every terminal. 5-minute cooldown between rotations.
Cookie
import
Zero-setup authentication via browser cookie import. Reads session cookies from Firefox (any platform) or Chrome (decrypts via macOS Keychain with PBKDF2 + AES-128-CBC). Your existing Google sessions flow straight into Playwright. No manual login. No browser:setup command unless you have neither browser.
WezTerm
grid
Dynamic 2-column pane layout from repos.conf. Each repo gets a pane running Claude Code with a background title updater showing repo · branch * · PR #123. Empirically-tuned split ratios for 2–5 rows. Configurable shortcuts via shortcuts.conf. skhd global hotkeys auto-generated on startup.
Dashboard
React + TypeScript web UI at localhost:3848. Shows active account, CLI auth status, monitor state. Live log stream via Server-Sent Events. One-click rotation, per-account switching, monitor start/stop. Dark theme matching the WezTerm color scheme.
Recovery
Claude Code slash command /auto-updater for OAuth repair. Five recovery procedures: CLI auth reset, browser session cleanup, full credential nuke, selector updates for UI changes, and WezTerm terminal re-auth. Symlinked into ~/.claude/commands/ by setup.
Rotation flow
Rate limit detected
Monitor daemon tails ~/.claude/debug/*.txt and catches HTTP 429, 529, "overloaded", "usage limit", etc.
CLI logout
claude auth logout clears the current CLI credentials.
Browser automation
Playwright launches with imported cookies, signs out of claude.ai, signs in with the next Google account via OAuth chooser.
URL interception
claude auth login runs with a BROWSER env trap. The OAuth URL is captured to a temp file and opened in the same Playwright session.
Callback
OAuth completes, redirects to localhost. CLI receives the token. State file updated.
Terminal re-auth
/login sent to all WezTerm panes running Claude via wezterm cli send-text. Desktop notification confirms the switch.
Configuration
accounts.conf
# one per line, rotates in order
alice@company.com
alice.personal@gmail.com
alice.backup@gmail.com
repos.conf
# directory=org/repo
myapp=myorg/myapp
backend=myorg/backend
frontend=myorg/frontend
shortcuts.conf
# directory=key (Opt+Cmd)
myapp=m
backend=b
frontend=f
WezTerm shortcuts
ShortcutAction
Opt Cmd `Toggle WezTerm visibility
Opt Cmd keyFocus repo pane (from shortcuts.conf)
Opt Cmd LFocus local shell pane
Opt Cmd POpen current repo's PR in browser
Opt Cmd NPush branch and open GitHub compare page
Opt Cmd RKill and restart Claude in current pane
Opt Cmd ↓Add a row of local panes
